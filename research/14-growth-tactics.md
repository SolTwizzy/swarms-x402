## 1. Lessons from Stripe & Twilio: What Worked in Their First 90 Days

### Stripe's "Collison Installation"
- Patrick Collison would literally install Stripe for developers on the spot during conversations. He'd say "can I see your laptop?" and set it up right there
- **Key insight**: They eliminated EVERY friction point. Developers could start accepting payments with 7 lines of code while competitors required weeks of paperwork
- They targeted YC companies and the SF developer community first -- tiny, high-quality cohort
- Word-of-mouth from happy early users (Shopify, Lyft) drove exponential growth
- Stripe took 2 years to get to 50 users, but those 50 were the RIGHT users

### Twilio's Developer-Centric Model
- Jeff Lawson's mantra: "treat developers like customers" -- sounds obvious but nobody did it in telecom
- They launched at conferences by handing out free API credits, not slides
- Twilio's famous launch at TechCrunch Disrupt: they gave every attendee a phone number to call, which triggered a live demo
- **Key insight**: Developers who USED the product became internal champions at companies (bottom-up adoption)
- The Stack Overflow Collective and developer community investment drove organic discovery

- [ ] **Target ElizaOS plugin users first**: They're already in the ecosystem, smallest distance to conversion

## 2. Product Hunt & Hacker News Launch Playbook

### What Works on Product Hunt
- **Timing**: Launch at 12:01 AM PT (Tuesday-Thursday are best)
- **Pre-launch**: Build a "hunter network" -- 50+ people who commit to upvoting and commenting on day 1
- **One Reddit user documented getting 3,565 upvotes**: Key was coordinating across PH, HN, BetaList, and curated directories simultaneously
- **Maker comments matter**: Respond to every single comment within minutes
- **GIF/video demo > screenshots**: Show the product working, not just describe it

### What Works on Hacker News (Show HN)
- **Title formula**: "Show HN: [What it does] in [impressive constraint]" -- e.g., "Show HN: Open-source tool that converts Figma designs to React in 30 seconds"
- Post between 6-9 AM ET for maximum visibility
- **Technical depth wins**: HN readers want to see the architecture, not marketing speak
- The "Show HN" tag is increasingly popular and front-page posts drive 10,000-20,000+ visits
- **Never be salesy**: Frame as "I built this, here's what I learned" not "buy my product"
- Allow reposting after initial post if first attempt didn't gain traction

## 3. Developer-First Go-to-Market for API Products

### The PLG (Product-Led Growth) Playbook
- **Self-serve first**: No signup walls, no sales calls. Developer tries it, it works, they pay
- **Documentation IS marketing**: Stripe's docs are legendary because they're the product's primary marketing channel
- **"Time to first API call" is your north star metric**: If it takes >5 minutes, you're losing 80% of developers
- **Free tier / generous sandbox**: Every successful API company has a free tier that's genuinely useful

### API-Specific Growth Hacks
- **Embed your API in tutorials**: Write "How to build X with SwarmX" tutorials that rank for long-tail keywords
- **SDK > raw API**: Developers who use your SDK have 3-5x higher retention than raw API users
- **Changelog as content**: Every feature update is a blog post, tweet, and community announcement
- **Status page transparency**: Public status pages build trust (use a free tool like Upptime on GitHub)

## 4. Open Source Marketing Playbook

### What Works for OSS in 2026
- **README is your landing page**: It should have a demo GIF, one-line install, and badges (build status, npm version, license)
- **"Awesome" lists**: Get listed on awesome-xyz lists relevant to your domain (awesome-ai, awesome-solana, awesome-agents)
- **Cross-post everywhere**: DEV.to, Hashnode, Medium -- same content, different audiences
- **Contributor-friendly issues**: Label issues "good first issue" and "help wanted" to attract contributors
- **CNCF/foundation strategy**: Not applicable now, but OSS foundations amplify visibility

### GitHub Stars Strategy
- Stars compound: each star makes the next one easier (social proof)
- README badges, clear install instructions, and a good "Why?" section drive stars
- **Star-for-star networks exist** but are low quality -- focus on genuine value
- Cross-promote from other repos via mentions in issues, PRs, and discussions

### Why Telegram Works for Crypto
- Crypto audiences already live on Telegram (400M+ MAU, heavily crypto-skewed)
- Telegram Mini Apps enable web app experiences inside Telegram itself
- Bot + group dynamics create viral loops: user tries bot -> shares in group -> friends try it

## 7. Getting on GitHub Trending

### How GitHub Trending Works
- Trending is based on **star velocity** (stars per unit time), not total stars
- Daily/weekly/monthly windows -- daily is easiest to hit
- Language-specific trending pages (TypeScript, JavaScript) are easier than "all languages"
- **Threshold**: ~30-50 stars in 24 hours can trend for TypeScript; ~100+ for "all languages"

### Tactics That Work
- **Coordinate launch day**: Get Show HN, PH, Twitter thread, and Reddit post all live on the same day
- **Cross-repo promotion**: If you have other repos with stars, add a mention/link to the new one
- **Conference/event timing**: Launch when a relevant conference is happening (developers are online and paying attention)
- **Release something genuinely useful**: Star-bait repos with just a README don't retain
- **Topic tags**: Add relevant GitHub topics (ai, agents, solana, x402, micropayments, multi-agent)

## 11. Referral Program Design

### What Works for API Products
- **Credits > cash**: Give API credits for referrals (costs you nothing until they're used)
- **Double-sided**: Both referrer and referee get credits (increases conversion)
- **Simple mechanics**: "Share your referral link, both get 100 free API calls"
- **Track via API key**: Each user's API key contains a referral code

### Successful Examples
- **Twilio**: $10 credit for referrer when referee spends $20+
- **DigitalOcean**: $200 credit for referrer, $100 for referee
- **Stripe**: Used word-of-mouth rather than formal referrals (product quality drove sharing)

- [ ] **Reward structure**: Referrer gets 50 free calls, referee gets 50 free calls

## Key Insight: The Compound Effect

Every tactic above reinforces the others:
- Show HN drives GitHub stars -> trending drives more stars -> more stars drive trust -> trust drives API adoption
- Blog posts rank on Google -> drive playground visits -> playground converts to API users -> users share on Twitter -> Twitter drives more blog views
- Telegram bot demonstrates value -> users visit playground -> playground leads to GitHub -> GitHub leads to ElizaOS plugin adoption

**The #1 mistake**: doing one channel at a time. The compounding effect only works when you launch across multiple channels simultaneously.

**The #1 lesson from Stripe/Twilio**: Make the product so easy to use that developers become evangelists without being asked.
